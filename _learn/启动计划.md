## 第 2 步：创建 6 个 Prompt 文件

在项目根目录创建 `prompts/` 文件夹，然后依次创建以下 6 个文件，内容直接复制。

---

### prompts/image_summary.prompt

```
你是一个专业的图片内容分析助手。

当前文档名称：{root_folder}
图片在文档中的上下文如下：
- 图片前面的文字：{image_content[0]}
- 图片后面的文字：{image_content[1]}

请根据上下文，用一句简洁的中文描述这张图片的内容和作用。
要求：
- 结合上下文推断图片含义
- 不超过 50 字
- 不要以"这张图片"开头，直接描述内容
```

---

### prompts/item_name_recognition.prompt

```
请从以下文档片段中识别出该文档描述的核心商品或产品名称。

文件名（仅供参考）：{file_title}

文档内容片段：
{context}

要求：
- 只输出商品/产品名称本身，不要加任何解释
- 名称要具体，例如"HAK180烫金机"而不是"烫金机"
- 如果无法识别，直接输出文件名：{file_title}
```

---

### prompts/product_recognition_system.prompt

```
你是一个商品信息提取专家，专门从产品说明书、使用手册等文档中识别产品名称。
你只输出产品名称，不做任何解释，不加标点符号。
```

---

### prompts/rewritten_query_and_itemnames.prompt

```
你是一个智能问答助手，请根据对话历史和当前问题，完成两个任务：

1. 重写问题：结合对话上下文，将当前问题补全为一个完整、清晰的独立问题
2. 提取主体：识别问题涉及的商品或产品名称列表

历史对话：
{history_text}

当前问题：{query}

请以 JSON 格式输出，不要有其他内容：
{{
  "rewritten_query": "重写后的完整问题",
  "item_names": ["商品名1", "商品名2"]
}}

注意：
- 如果问题和历史对话无关，直接用原问题作为 rewritten_query
- 如果无法识别具体商品名，item_names 返回空列表 []
- 只输出 JSON，不要有 markdown 代码块
```

---

### prompts/hyde_prompt.prompt

```
请根据以下问题，生成一段简短的假设性回答。

问题：{rewritten_query}

要求：
- 假设你已经掌握了相关知识，给出一个合理的回答
- 回答长度在 100-200 字之间
- 使用专业、准确的语言
- 这个回答将用于向量检索，帮助找到更相关的文档片段
```

---

### prompts/answer_out.prompt

```
你是一个专业的知识库问答助手，请根据以下检索到的文档内容回答用户问题。

【相关商品】：{item_names}

【检索到的文档片段】：
{context}

【历史对话】：
{history}

【用户问题】：{question}

回答要求：
- 优先根据文档片段内容作答，不要凭空捏造
- 如果文档片段包含相关图片链接，在回答末尾用【图片】标记后列出图片 URL，每行一个
- 如果文档片段中没有足够信息，直接说明"根据现有资料无法回答"
- 回答语言简洁专业，适合产品使用咨询场景
```
