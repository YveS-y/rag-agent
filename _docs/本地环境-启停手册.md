# 本地环境启停手册

记录本机开发环境的启动/停止流程，以及当前项目的整体进度，方便中断后快速恢复。

## 〇、一键脚本（推荐）

```bash
# 启动：Docker(Milvus/MinIO/MongoDB) + 导入服务(8000) + 查询服务(8001)
./scripts/start.sh

# 停止：先停两个 Python 服务，再 docker compose stop（数据保留）
./scripts/stop.sh
```

下面的章节是脚本背后对应的手动操作步骤，脚本出问题时可以参考手动排查。

## 一、依赖的基础设施（Docker）

`docker-compose.yml` 定义了 4 个容器：etcd、Milvus 内部依赖的 MinIO、Milvus standalone、业务用 MinIO、MongoDB（共 5 个服务：etcd / milvus-minio / milvus-standalone / minio / mongo）。

**启动：**
```bash
cd /Users/ys/code/rag-agent
docker compose up -d
```

**查看状态：**
```bash
docker compose ps
```

**停止（保留数据）：**
```bash
docker compose stop
```

**完全停止并删除容器（数据卷 `./volumes/` 仍保留，下次 up 会自动恢复数据）：**
```bash
docker compose down
```

**端口对照：**
| 服务 | 端口 | 用途 |
|------|------|------|
| Milvus standalone | 19530 | 向量数据库 |
| 业务 MinIO | 9000 (API) / 9001 (控制台) | 文件/图片存储，账号密码 minioadmin/minioadmin |
| MongoDB | 27017 | 对话历史 |

首次启动后需确认 MinIO 里 `rag-bucket` 桶存在（已建好，重新 `docker compose down` 不会删除数据卷里的桶）。

## 二、环境切换（本地 / 云端）

`.env` 里的 `APP_ENV` 决定加载哪个基础设施配置：
- `APP_ENV=local` → 加载 `.env.local`（本机 Docker 地址）
- `APP_ENV=cloud` → 加载 `.env.cloud`（云端服务器地址，**目前 MinIO 密钥还没填，云端暂不可用**）

切换只需改 `.env` 这一行，不需要改代码。

## 四、启动业务服务

两个独立的 FastAPI 服务，分别负责导入和查询：

```bash
# 文档导入服务（端口 8000）
uv run python -m app.import_process.api.import_server

# 查询问答服务（端口 8001）
uv run python -m app.query_process.api.query_server
```

**对应页面：**
- 导入页：http://127.0.0.1:8000/import
- 聊天页：http://127.0.0.1:8001/chat.html

**停止：** 直接 Ctrl+C，或 `kill <pid>`。

## 五、健康检查

```bash
curl http://127.0.0.1:8000/docs        # 导入服务存活
curl http://127.0.0.1:8001/health      # 查询服务存活
```
