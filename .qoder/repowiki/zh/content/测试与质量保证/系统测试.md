## 引言
本文件面向系统测试工程师与开发者，系统化梳理并文档化以下测试能力与验证方法：
- 环境变量优先级测试：验证系统环境变量与配置文件（.env）的优先级规则，并给出可复现实验步骤。

- 图形流程测试：验证导入与查询主流程图的执行与状态转换，提供端到端验证方案。
- 系统集成测试：给出跨模块（配置、日志、流程图）的集成验证步骤与场景设计。
- 性能基准与压力测试：提供可落地的基准与压力测试实施建议。
- 结果分析与问题诊断：建立测试结果分析与问题定位流程。

**更新** 本版本新增两阶段架构说明，涵盖导入阶段（.pdf和.md文件处理）与查询阶段（三路召回）的完整流程，为系统测试提供架构层面的参考。

### 环境变量优先级测试
目标：验证系统环境变量对.env的覆盖优先级；演示如何让.env覆盖系统变量。
- 测试入口：test/01-env和系统环境变量的优先级.py
- 关键点：
  - 默认行为：load_dotenv() 不覆盖已有系统环境变量，系统变量优先级更高。
  - 覆盖行为：传入 override=True 显式允许.env覆盖系统变量。
- 验证步骤（建议）：
  1) 在系统设置 OPENAI_API_KEY=system_val，同时在 .env 设置 OPENAI_API_KEY=dotenv_val。
  2) 运行测试脚本，观察 OPENAI_API_KEY 的读取结果（应为 system_val）。
  3) 将脚本改为 load_dotenv(override=True)，再次运行，观察 OPENAI_API_KEY 的读取结果（应为 dotenv_val）。
- 影响范围：所有配置模块（如 LLMConfig）均通过 os.getenv 读取，受上述优先级影响。

### 图形流程测试（导入与查询）

#### 两阶段架构概述
**更新** RAG项目采用两阶段架构设计：

**导入阶段（Import Phase）**：
- 接收 .pdf 和 .md 文件，其他格式报错
- PDF 文件通过 MinerU 服务转换为 Markdown
- 图片内容转换为说明并存储到 MinIO 对象存储
- 文本内容和图片说明存入 Milvus 向量数据库
- 历史会话记录存入 MongoDB
- 知识图谱存入 Neo4j 数据库

**查询阶段（Query Phase）**：
- 三路召回机制：向量化召回、假设性文档嵌入（HyDE）召回、Web 搜索召回
- 通过 reciprocal rank fusion（RRF）算法进行结果融合排序
- 最终格式化答案输出

#### 导入阶段节点详解
**更新** 基于测试.txt中的架构说明，导入阶段包含以下关键节点：

1. **node_pdf_to_md**：PDF 转 Markdown 节点
   - 使用 MinerU 服务进行 PDF 解析
   - 支持轮询获取解析结果
   - 下载并解压 Markdown 文件

2. **node_md_img**：Markdown 图片处理节点
   - 识别 Markdown 中的图片引用
   - 上传图片到 MinIO 对象存储
   - 生成图片描述并替换链接

3. **node_document_split**：文档切分节点
   - 基于标题层级进行语义切分
   - 处理过长段落的二次切分
   - 合并过短切片以优化检索效果

4. **node_item_name_recognition**：主体识别节点
   - 使用 LLM 识别文档核心商品名称
   - 生成向量并存储到 Milvus
   - 支持幂等性清理

5. **node_bge_embedding**：向量化节点
   - 使用 BGE-M3 模型生成稠密和稀疏向量
   - 批量处理优化性能

6. **node_import_milvus**：Milvus 导入节点
   - 创建集合并配置索引
   - 批量插入向量数据
   - 支持按商品名称删除旧数据

#### 查询阶段节点详解
**更新** 查询阶段包含以下关键节点：

1. **node_search_embedding**：向量检索节点
   - 混合查询 Milvus 向量数据库
   - 支持稠密和稀疏向量检索
   - 返回相关文档片段

2. **node_search_embedding_hyde**：HyDE 检索节点
   - 生成假设性文档并进行向量检索
   - 提高检索召回率
   - 支持商品名称过滤

3. **node_web_search_mcp**：Web 搜索节点
   - 调用百炼 MCP 服务进行网络搜索
   - 获取外部知识补充
   - 支持流式 HTTP 调用

4. **node_rrf**：结果融合节点
   - 使用 reciprocal rank fusion 算法

## 故障排查指南
- 环境变量未生效：
  - 确认 .env 文件路径与权限，检查 load_dotenv() 是否在读取 os.getenv 前执行。
  - 使用 override=True 显式允许 .env 覆盖系统变量。
- 日志未输出或级别不符：
  - 检查 .env 中 LOG_CONSOLE_ENABLE/LOG_FILE_ENABLE 与 LOG_CONSOLE_LEVEL/LOG_FILE_LEVEL。
  - 确认 init_logger() 已被导入模块调用，且未被二次 remove。
- CUDA 无法使用：
  - 确认已安装匹配的 PyTorch 与 CUDA 运行时；在 CPU 环境下 CUDA 状态为 False 属正常。
- 流程图卡住或状态异常：
  - 通过流式输出逐节点定位；检查状态字段是否按预期更新；核对条件边路由逻辑。
- 异常未记录或堆栈缺失：
  - 使用 logger.exception 或在 except 块中记录异常并携带堆栈。
