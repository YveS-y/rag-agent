## 四、学习计划（面试准备）

> 对应 `docs/项目要总结内容.txt` 的 7 个方向

### 4.1 RAG 的概念和理解

**要能用一句话解释**：RAG 是在 LLM 生成答案前，先从外部知识库检索相关片段，将其拼入 prompt，让模型基于事实作答，而非依赖训练记忆。

**核心价值**：
- 解决知识截止问题（私域/实时数据）
- 减少幻觉（有来源可引用）
- 无需微调即可更新知识

---

### 4.2 RAG 宏观步骤

```
离线阶段：文档 → 切块 → 向量化 → 存入向量库
在线阶段：用户问题 → 向量化 → 检索 → 重排 → LLM 生成 → 返回答案
```

---

### 4.3 本项目的 RAG 步骤

**导入链路**（LangGraph 7节点）：
```
PDF → MineRU解析 → Markdown
    → 产品名识别（item_name）
    → 文档切块（normal / advanced）
    → BGE-M3 双路向量（dense + sparse）
    → 写入 Milvus
    → （可选）图片上传 MinIO
```

**查询链路**（LangGraph）：
```
用户问题
  → 产品名确认（item_name_confirm）
  → 标准向量检索（search_embedding）
  → HyDE 假设文档检索（search_embedding_hyde）
  → RRF 融合排序（node_rrf）
  → BGE-Reranker 重排
  → （可选）MCP Web 搜索补充
  → LLM 生成答案（SSE 流式）
  → 保存对话历史 MongoDB
```

---

### 4.4 保证召回准确的 5 个手段

| # | 手段 | 本项目实现 |
|---|------|-----------|
| 1 | **混合检索**（dense + sparse） | BGE-M3 同时输出两路向量，Milvus ANN + BM25 |
| 2 | **HyDE**（假设文档扩展） | `node_search_embedding_hyde.py` |
| 3 | **RRF 融合**（多路结果合并） | `node_rrf.py` |
| 4 | **Reranker 精排** | BGE-Reranker-Large，`node_rerank.py` |
| 5 | **产品名过滤**（精确元数据筛选） | `node_item_name_confirm.py` 过滤 Milvus expr |

### 4.5 核心技术栈（面试口头表达模板）

> "这个项目用 **LangGraph** 编排多步骤工作流，向量化用 **BGE-M3** 生成 dense+sparse 双路向量存入 **Milvus**，检索后用 **BGE-Reranker** 精排，LLM 用 **Qwen3** 通过 DashScope 的 OpenAI 兼容接口调用，PDF 解析用 **MineRU**，文件存 **MinIO**，对话历史存 **MongoDB**，整体用 **FastAPI** 对外提供 SSE 流式接口。"

### 4.7 RAG 面试高频问题 & 答题框架

**Q1: 你们 RAG 准确率怎么保证？**
> 多路召回（dense+sparse+HyDE）→ RRF 融合 → Reranker 精排，P3 阶段会补充自动化评估数字。

**Q2: 向量维度多少？切块策略是什么？**
> BGE-M3 dense 1024 维，sparse 维度动态；切块按 500 token，overlap 50 token，长表格用高级切块保留结构。

**Q3: 为什么用 LangGraph 而不是直接写代码？**
> 状态机管理多步骤流程，节点可独立测试，状态流转可视化，支持条件分支（如 HyDE 失败降级）。

**Q4: HyDE 是什么？有什么风险？**
> 用 LLM 生成"假设的答案文档"再向量化去检索，好处是语义对齐更好；风险是 LLM 幻觉的假设文档可能跑偏，需加日志监控是否命中。

**Q5: Milvus 混合检索怎么做的？**
> 同一 Collection 存 dense vector（HNSW 索引）和 sparse vector（稀疏反转索引），查询时两路并行 ANN，再用 RRF 合并排名。

## 五、踩坑预警（开发中注意）

| # | 坑 | 解法 |
|---|----|------|
| 1 | `.venv` 含绝对路径，移动项目后失效 | 重新 `uv sync` |
| 2 | Docker 显示 running 但 Milvus 未就绪 | 启动后等 20-30 秒再连接 |
| 3 | VPN Fake IP 模式拦截 localhost:19530 | 关闭 VPN 或切换为真实 IP 模式 |
| 4 | 新增 YAML 配置字段但 dataclass 未同步 | 改完 YAML 同步改对应 `conf/*.py` |
| 5 | 日期存 int（yyyyMMdd）LLM 无法正确处理 | 改为 ISO datetime 格式 |
| 6 | 换 embedding 模型后维度不匹配 | 删除重建 Milvus Collection |
| 7 | BGE-M3 初次加载 1-3 分钟无输出 | 看 CPU 占用，不要误判为卡死 |
| 8 | HyDE 失败静默降级，不知道是否生效 | 在 `node_search_embedding_hyde.py` 加日志 |
